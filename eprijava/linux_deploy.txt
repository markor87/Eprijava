ПРЕЛАЗАК НА LINUX СЕРВЕР
========================

1. NGINX КОНФИГУРАЦИЈА
----------------------
Направи фајл /etc/nginx/sites-available/eprijava:

    server {
        listen 80;
        server_name твоj-домен.rs;
        root /var/www/eprijava/public;

        index index.php;

        location / {
            try_files $uri $uri/ /index.php?$query_string;
        }

        # Засебан блок за SSE руту — мора бити ИЗНАД општег location ~ \.php$ блока
        # fastcgi_buffering off спречава буферисање и ERR_HTTP2_PROTOCOL_ERROR
        location /generate-competition-zip {
            include snippets/fastcgi-php.conf;
            fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
            fastcgi_read_timeout 7200;
            fastcgi_buffering off;
            fastcgi_request_buffering off;
        }

        location ~ \.php$ {
            include snippets/fastcgi-php.conf;
            fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
            fastcgi_read_timeout 3600;
        }

        location ~ /\.ht {
            deny all;
        }
    }

Потом:
    sudo ln -s /etc/nginx/sites-available/eprijava /etc/nginx/sites-enabled/
    sudo nginx -t
    sudo systemctl reload nginx


2. PHP-FPM КОНФИГУРАЦИЈА
------------------------
Отвори /etc/php/8.2/fpm/pool.d/www.conf и провери/постави:

    request_terminate_timeout = 0

Ако је подешено на нпр. 300, повећај или стави 0 (без лимита).
Ово је важно да PHP не прекида генерисање ZIP-а за велики број пријава.

Рестарт:
    sudo systemctl restart php8.2-fpm


3. ДОЗВОЛЕ НА ФОЛДЕРИМА
------------------------
    sudo chown -R www-data:www-data /var/www/eprijava
    sudo chmod -R 755 /var/www/eprijava
    sudo chmod -R 775 /var/www/eprijava/storage
    sudo chmod -R 775 /var/www/eprijava/bootstrap/cache

Фолдер storage/app/downloads се прави аутоматски при генерисању ZIP-а,
али мора да постоји права дозвола на storage/app/ (обухваћена горе).


4. .ENV ПОДЕШАВАЊА
------------------
Копирај .env.example у .env и постави:

    APP_ENV=production
    APP_DEBUG=false
    APP_URL=https://твоj-домен.rs

    DB_HOST=127.0.0.1
    DB_DATABASE=eprijava
    DB_USERNAME=...
    DB_PASSWORD=...

    FILESYSTEM_DISK=local
    SESSION_DRIVER=file     # или database/redis ако постоји
    CACHE_STORE=file        # или database/redis ако постоји

    QUEUE_CONNECTION=database   # обавезно — е-маил потврде пријаве и безбедносни аларми иду кроз ред


5. ИНСТАЛАЦИЈА И ПОКРЕТАЊЕ
---------------------------
    cd /var/www/eprijava
    composer install --no-dev --optimize-autoloader
    php artisan key:generate
    php artisan migrate --force
    php artisan storage:link
    php artisan config:cache
    php artisan route:cache
    php artisan view:cache


6. ПРИВАТНИ ФАЈЛОВИ (сертификати)
----------------------------------
Сертификати се чувају у storage/app/ преко local диска.
То је ван public/ фолдера и нису директно доступни преко веба — OK.
Постоји /private-files/{path} рута са auth мидлвером за служење.


7. HTTPS (препоручено)
----------------------
    sudo apt install certbot python3-certbot-nginx
    sudo certbot --nginx -d твоj-домен.rs

Certbot ће сам допунити nginx конфигурацију.


8. ТЕСТИРАЊЕ SSE (Преузимање пријава)
--------------------------------------
После деплоја, кликни Преузми пријаве и провери да ли бројач расте у реалном
времену. Ако бројач стоји на 0 па одједном скочи на 100%, или ако добијаш
ERR_HTTP2_PROTOCOL_ERROR у конзоли — провери да ли је location блок за
/generate-competition-zip из секције 1 исправно додат и да ли је nginx рестартован.


9. APACHE2 (алтернатива уместо nginx)
--------------------------------------
Ако користиш Apache2 уместо nginx:

а) Онемогући буферисање одговора (важно за SSE) у .htaccess или VirtualHost:

    SetEnv no-gzip 1
    SetEnv dont-vary 1

б) Постави таjмаут у VirtualHost конфигурацији:

    TimeOut 7200

в) Ако користиш Apache + PHP-FPM преко mod_proxy_fcgi, исто важи:
    request_terminate_timeout = 0  (у /etc/php/8.2/fpm/pool.d/www.conf)

г) Ако користиш mod_php уместо FPM, set_time_limit(0) у PHP коду је довољан —
   нема посебне FPM конфигурације.

д) Ако је укључен mod_http2 и добијаш ERR_HTTP2_PROTOCOL_ERROR у конзоли,
   онемогући HTTP/2 само за SSE руту:

    <Location /generate-competition-zip>
        Protocols http/1.1
    </Location>

е) HTTPS са Let's Encrypt за Apache:

    sudo apt install certbot python3-certbot-apache
    sudo certbot --apache -d твоj-домен.rs


10. БЕЗБЕДНОСНА УПОЗОРЕЊА (логовање пријава, е-маил аларми)
------------------------------------------------------------

а) Додај у .env:

    SECURITY_ALERT_EMAIL=admin@твоj-домен.rs
    # QUEUE_CONNECTION=database је већ наведено у секцији 4

б) TRUSTED PROXIES — обавезно ако је апликација иза nginx/Apache!
   Без овога request()->ip() враћа 127.0.0.1 уместо стварне IP адресе
   корисника, па се геолокација и провера нове IP адресе никада не покрећу.

   Отвори bootstrap/app.php и додај у withMiddleware:

       ->withMiddleware(function (Middleware $middleware): void {
           $middleware->trustProxies(at: '*');
       })

   Или постави у .env (ако верзија Laravела подржава):

       APP_TRUSTED_PROXIES=*

в) QUEUE WORKER — обавезан! Без покренутог воркера е-маил потврда приликом
   подношења пријаве и безбедносни е-маил аларми никад се не шаљу —
   послови чекају у табели jobs и никад се не извршавају.

   Инсталирај Supervisor:
       sudo apt install supervisor

   Направи /etc/supervisor/conf.d/eprijava-worker.conf:

       [program:eprijava-worker]
       process_name=%(program_name)s_%(process_num)02d
       command=php /var/www/eprijava/artisan queue:work --sleep=3 --tries=3 --max-time=3600
       autostart=true
       autorestart=true
       stopasgroup=true
       killasgroup=true
       user=www-data
       numprocs=1
       redirect_stderr=true
       stdout_logfile=/var/log/eprijava-worker.log
       stopwaitsecs=3600

   Покрени:
       sudo supervisorctl reread
       sudo supervisorctl update
       sudo supervisorctl start eprijava-worker:*

г) После деплоја покрени миграцију (прави табелу login_logs и jobs):
       php artisan migrate --force

   Табела jobs се прави аутоматски ако не постоји:
       php artisan queue:table
       php artisan migrate --force


11. SCHEDULER (аутоматско брисање ZIP фајлова)
-----------------------------------------------
Апликација чува ZIP архиве у storage/app/downloads/ и брише их после успешног
преузимања. Scheduler команда чисти евентуалне заостале фајлове старије од 24 сата.

Додај у crontab (sudo crontab -e -u www-data):

    * * * * * cd /var/www/eprijava && php artisan schedule:run >> /dev/null 2>&1

Scheduler покреће cleanup сваки дан у 03:00.

Провера:
    php artisan schedule:list
